Astro

High performance SQL layer over HBase by using Spark SQL framework.

Welcome to Astro.

Astro is fully distributed SQL engine on HBase by leveraging Spark ecosystem. It enables systematic and powerful handling of data pruning, intelligent scan, and pushdowns like custom filters and coprocessor, and make more traditional RDBS capabilities possible.

Why Astro

HBase is a very useful big data store but its access mechanism is very primitive and only through client-side APIs, Map/Reduce interfaces and interactive shells. Astro provides SQL layer over Hbase, so it takes your SQL query, compiles it into optimized Spark plan and in turn it does series of HBase scans to result sets.User do not require to write any complex client code to access HBase tables.

For example,to count rows in Hbase table we need to write following code.

public class MyAggregationClient {
    private static final byte[] TABLE_NAME = Bytes.toBytes("CNCC_Demo");
    private static final byte[] CF = Bytes.toBytes("f1");
    public static void main(String[] args) throws Throwable {
        Configuration customConf = new Configuration();
        customConf.setStrings("hbase.zookeeper.quorum","node0,node1,node2");
        customConf.setLong("hbase.rpc.timeout", 600000);
        customConf.setLong("hbase.client.scanner.caching", 1000);
        Configuration configuration = HBaseConfiguration.create(customConf);
        AggregationClient aggregationClient = new AggregationClient(configuration);
        Scan scan = new Scan();        scan.addFamily(CF);
        long rowCount = aggregationClient.rowCount(TABLE_NAME, null, scan);
        System.out.println("row count is " + rowCount);    }}

But if we use Astro just create table schema and fire the SQL query.

select  count(*)  from CNCC_Demo;

CREATE TABLE CNCC_Demo (
      index          INTEGER,
      event_time     STRING,
      country        STRING,
      site           STRING,
      PRIMARY KEY(index))
      MAPPED BY (CNCC_Demo, COLS=   [event_time=f1.c1,country=f1.c2,site=f2.c1]);
